你的任务是在当前工作目录下开发一个 Python Web 工具。

## 用户需求
请帮我开发一个“AI 图像描边/贴纸生成工具”的 Web 应用。
### 1. 需要维护的核心数据（状态）
- 原始图片：存储用户上传的图片数据。
- 处理后的图片：存储 AI 返回的生成结果。
- 处理状态：布尔值，记录当前是否正在请求 API 处理中。
- 错误信息：字符串，用于展示请求失败或格式错误的提示。
- 提示词 (Prompt)：多行文本，需提供默认值（大致内容为：识别主体并添加白色描边，背景纯色或透明）。

### 2. 页面结构与功能区划分
页面分为顶部导航、核心工作区和底部说明区。

#### 2.1 核心工作区：输入与控制（左侧/上方区域）
1. **图片上传区**：
   - **交互**：支持点击区域弹出文件选择框，同时支持将图片文件直接拖拽到该区域进行上传。
   - **空状态**：显示“点击上传或拖拽图片”的提示文案和图标。
   - **已上传状态**：显示原图预览。当鼠标悬浮在预览图上时，需显示一个半透明遮罩层和“更换图片”的提示，点击可重新上传。
   - **逻辑**：读取到图片后，需将其转换为 Base64 格式保存，并清空历史的处理结果和错误信息。

2. **控制面板**：
   - **提示词输入框**：一个支持多行输入的文本域，用户可修改默认提示词。
   - **错误提示区**：仅在“错误信息”非空时显示。
   - **操作按钮**：
     - **禁用状态**：当没有上传图片或正在处理中时，按钮不可点击。
     - **默认状态**：显示“开始处理”。
     - **处理中状态**：显示 Loading 动画和“正在处理”文案。点击按钮触发 API 请求逻辑。

#### 2.2 核心工作区：结果展示（右侧/下方区域）
1. **结果区头部**：
   - 包含标题“处理结果”。
   - **下载按钮**：仅当“处理后的图片”存在时才显示。点击后，触发浏览器下载该处理后的图片（文件名为特定名称如 `sticker.png`）。
2. **结果预览区**：
   - **空状态**：默认显示占位图标及“处理后的图片将显示在这里”。
   - **处理中状态**：显示居中的 Loading 动画及“正在识别主体轮廓...”的提示。
   - **完成状态**：展示 API 返回的处理后的图片。

#### 2.3 底部说明区
- 静态文本区域，说明 AI 图像生成的随机性，提示用户如果效果不好可以修改提示词重试。

### 3. 核心 API 请求逻辑 (模拟大模型生图)
当用户点击“开始处理”按钮时：
1. 开启“处理状态”（触发界面 Loading）。
2. 将当前的“原始图片 (Base64)”和“提示词”作为参数，构造请求发送给后端的 AI 生图接口。
3. **成功响应**：解析接口返回的数据，提取生成的图片 Base64，更新到“处理后的图片”中展示。
4. **异常处理**：如果接口报错、返回格式不对（例如只返回了文本而没有图片），需要捕获异常，将具体的报错原因更新到“错误信息”中展示。
5. 无论成功还是失败，最后都必须关闭“处理状态”。

请根据以上需求，完整实现。

## 强制规则（违反任何一条即为失败）

### 文件规范
1. 将最终可运行的代码写入当前目录的 `main.py`，必须是单文件 FastAPI 应用。
2. 文件开头必须包含 PEP 723 inline metadata：
   ```
   # /// script
   # requires-python = ">=3.10"
   # dependencies = ["fastapi", "uvicorn"]
   # ///
   ```
   将 dependencies 替换为你实际使用的所有第三方库。

### 运行规范
3. 动态端口：使用 `int(os.environ.get("PORT", 8000))`，禁止硬编码。
4. 绑定地址：仅 `127.0.0.1`，禁止 `0.0.0.0`。
5. 入口点：文件末尾必须是：
   ```python
   if __name__ == "__main__":
       import uvicorn
       uvicorn.run(app, host="127.0.0.1", port=int(os.environ.get("PORT", 8000)))
   ```

### URL 规范
6. 本应用将被反向代理挂载在子路径 `/tools/ai_sticker_generator/` 下。
   HTML/JS 中所有 fetch/XHR/form URL 必须是相对路径（无前导 `/`）。
   使用 `api/upload` 而非 `/api/upload`。

### UI 设计规范 (VibeHub 统一风格)
7. 颜色：主色 #cba186，背景 #f0f2f5，白色卡片
8. 卡片：border-radius: 16px, box-shadow: 0 2px 12px rgba(0,0,0,.08)
9. 按钮：主按钮 #cba186，次按钮 #eee，下载按钮 #000，圆角 10px
10. 布局：Header + 上传卡片 + 按钮行 + Grid 预览
11. 响应式：桌面 3 列，平板 2 列，手机 1 列
12. 所有 UI 文本使用中文

### 自测流程
13. 写完 `main.py` 后，你**必须**自己验证代码可运行：
    - 使用 Bash 工具执行：`PORT=18999 E:/Such_Proj/Other/VibeHub/bin/uv.exe run main.py &`（后台启动）

    - 测试完成后，务必 kill 掉后台进程
    - 最多重试 3 次
